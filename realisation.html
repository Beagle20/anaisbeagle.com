Proxy filtrant Squid — Conteneur cab1 / BTS SIO

Projet E6 BTS SIO SISR — Proxy Squid dans un conteneur systemd-nspawn

Objectif du projet

Dans le cadre du BTS SIO option SISR, j'ai déployé un proxy filtrant Squid à l'intérieur d'un conteneur systemd-nspawn (cab1) hébergé sur mon PC2 Debian 12 en salle E63. Le conteneur dispose d'une adresse IP fixe (10.63.101.70), garantissant une adresse proxy stable pour tous les postes du réseau. Le proxy authentifie obligatoirement les utilisateurs et applique des règles de filtrage différenciées selon le profil. L'ensemble est réalisé en ligne de commande en tant que projet2, superutilisateur du conteneur.

Bilan du projet

Le proxy filtrant Squid est opérationnel dans le conteneur cab1 (10.63.101.70:3128), accessible depuis tous les postes du réseau BTS. L'authentification Basic fonctionne (407 sans auth), le filtrage différencié est prouvé par les codes HTTP (403/301/302), et le chaînage vers le Tjener est confirmé par les en-têtes Via et par FIRSTUP_PARENT/10.0.2.2 dans les logs.

Limite identifiée : Squid ne filtre pas le HTTPS nativement — la solution serait le SSL Bump (hors périmètre de ce projet).
