Infrastructure Linux — Entreprise TechNova

TP BTS SIO SISR — Administration Linux Debian 12

Objectif du TP

J'ai déployé une infrastructure réseau complète sur mon poste Debian 12 en simulant l'environnement informatique de l'entreprise TechNova. L'objectif était de configurer et administrer plusieurs services réseau en ligne de commande, gérer des utilisateurs avec des droits différenciés, et rendre l'ensemble accessible via un nom de domaine interne.

Services déployés

Apache2 — Port 80 vsftpd FTP — Port 21 BIND9 DNS — Port 53 SSH — Port 22

Architecture réseau

Mon PC — Debian 12

10.0.26.81

Interface : pont1 (bridge)

Domaine : technova.local

Services : Apache2, vsftpd, BIND9, SSH

Switch réseau

10.0.0.1

Rôle : Commutation LAN

Connecté à : Mon PC + Autre PC

Tjener (passerelle)

10.0.2.2

Rôle : Passerelle vers Internet

Connecté via : le switch LAN

Gestion des utilisateurs et des droits

A
alice UID 1005 — groupes : alice, users, admin, dev
✓ FTP /home/alice ✓ Écrit /srv/dev ✓ SSH
B
bob UID 1006 — groupes : bob, users, compta
✓ FTP /home/bob ✗ /srv/dev refusé ✓ SSH

Commandes clés utilisées

# Création des groupes
groupadd admin
groupadd dev
groupadd compta

# Création des utilisateurs
adduser alice
adduser bob

# Attribution des groupes
usermod -aG dev alice
usermod -aG admin alice
usermod -aG compta bob

# Permissions sur /srv/dev
chown root:dev /srv/dev
chmod 770 /srv/dev

# DNS — test de résolution
nslookup web.technova.local 127.0.0.1

# Vérification des services
ss -tulpn | grep 80
ss -tulpn | grep 21
ss -tulpn | grep 22
ss -tulpn | grep 53

Résolution du domaine technova.local

J'ai utilisé deux mécanismes complémentaires pour que http://technova.local fonctionne :

BIND9 (DNS interne)

Résout technova.local pour toutes les machines du réseau local qui interrogent mon DNS.

/etc/hosts (local)

Ligne ajoutée : 10.0.26.81 technova.local — prioritaire sur le DNS, garantit l'accès local.

Bilan du projet

Ce TP m'a permis de déployer une infrastructure Linux complète et cohérente en ligne de commande. J'ai configuré quatre services réseau, géré les utilisateurs et leurs droits selon le principe du moindre privilège, et mis en place un domaine interne technova.local accessible par son nom grâce au DNS BIND9 et au fichier /etc/hosts.

Résultat clé : http://technova.local s'affiche dans le navigateur — preuve que le DNS, Apache2, la gestion des droits et les comptes utilisateurs fonctionnent tous ensemble de bout en bout.
